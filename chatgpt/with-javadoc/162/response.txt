Sure, here's an example refactoring code that replaces the use of the deprecated `getLoopbackMode()` method with `getOption()` and `StandardSocketOptions.IP_MULTICAST_LOOP`:

Before:

```
MulticastSocket multicastSocket = new MulticastSocket(port);
boolean loopbackMode = multicastSocket.getLoopbackMode();
System.out.println("Loopback mode: " + loopbackMode);
```

After:

```
MulticastSocket multicastSocket = new MulticastSocket(port);
boolean loopbackMode = multicastSocket.getOption(StandardSocketOptions.IP_MULTICAST_LOOP);
System.out.println("Loopback mode: " + loopbackMode);
```

As you can see, the `getOption()` method is used to retrieve the value of the `IP_MULTICAST_LOOP` socket option, which replaces the use of the deprecated `getLoopbackMode()` method.
